An alert is one kind of assertive aria-live region, and a slightly more urgent form of an status, which is a polite aria-live region. It is an important message, usually with time-sensitive information, that is read by the screen reader as soon as content is inserted into the alert element. The alert does not move the user's keyboard focus because there are usually no interactive components in an alert message, but the JavaScript programmer may choose to send the focus to the alert message, if appropriate.

By way of contrast, an alert dialog requires some user interaction, so the focus needs to go to the alert dialog.

Purposes of an alert can include:

Information alerts
Note: you could use status instead of alert for less critical messages.
Error alerts
When the user or the system does something wrong
Warning alerts
To prevent the user from making bad choices, or to at least be sure the user is aware of the consequence of an action
Success alerts
To let users know that their actions were processed correctly

Simple alert Example

When users click on this button, an alert message will appear below the button, and screen readers will read the alert message text. The focus will not move from the button.

HTML Markup:


<button id="savePrefs">Save my preferences</button>
<div id="savePrefsMsg" class="msg" role="alert"></div>

JavaScript Code:


<script>
$("#savePrefs").click(function() {
$("#savePrefsMsg").addClass("alert").text("Your preferences have been saved.");
return false;
});
</script>

Styles


.alert {
    margin:20px 40px;
    padding:20px;
    border:2px solid #990000;
    background:#fefee7;
    border-radius:10px
}

Example: Shopping Cart Update with alert and alertdialog

When you change options (update, delete, or save for later) in the shopping cart below, an alert message displays below the table, and is read by the screen reader. The keyboard focus does not go to the alert message. The delete button is slightly different, though. Instead of only activating the alert below the table, the delete button brings up a popup modal alert dialog and forces users to either confirm that they want to delete, or cancel. The keyboard focus goes to this modal alertdialog. Once users make this selection, the focus returns to the original delete button and the alert message below the table is updated.
